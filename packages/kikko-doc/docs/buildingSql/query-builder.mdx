# Query builder


We all know that, more often than not, query builders suck. Usually it is because they try to support too many SQL dialects.
But, with just one dialect, we managed to make a builder that is flexible enough, and supports all the possible SQLite queries.

Also, it is fully immutable. When we make a function call, it returns a new object instance. It is especially helpful when we are building composable queries.

And if query builder functionality is not enough, we could always use [raw queries](raw) and even embed them into query builder. When we use
raw builder our queries still remain reactive (but we need to mark tables with `sql.table('tableName')`).

For now, it still lacks some functionality like create table/create trigger. We plan to add support for them too.

## Installation

```bash
npm install @kikko-land/query-builder --save
```

Or

```bash
yarn add @kikko-land/query-builder
```
