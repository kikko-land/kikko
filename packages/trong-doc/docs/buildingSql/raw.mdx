# Raw

The idea is taken from [sql-template-tag](https://github.com/blakeembrey/sql-template-tag), big thanks to the author! It uses the power of ES2015 tagged template string for preparing SQL statements. Actually the whole query builder build on top of the `@trong-orm/sql`.

When you using raw queries, you need to follow one rule to keep your queries reactive:

:::info
Always use `sql.table('tableName')` when you reference to the table in the query. It allows too keep your queries reactive.

```typescript
import { sql } from "@trong-orm/sql";

const booksTables = sql.table("books");
sql`SELECT * FROM ${booksTables}`;
sql`INSERT INTO ${booksTables} VALUES (1, 2, 3)`;
```
:::

## Installation

```bash
npm install @trong-orm/sql --save
```

Or

```bash
yarn add @trong-orm/sql
```
